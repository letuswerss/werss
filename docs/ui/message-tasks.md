# 消息任务页面

消息任务页用于把“文章数据”主动推送出去。它不是内容生产页，而是自动分发页，适合把公众号更新定时发到企业微信、钉钉、飞书或其他 Webhook 接收端。

## 1. 访问路径

- 列表页：`/message-tasks`
- 新建页：`/message-tasks/add`
- 编辑页：`/message-tasks/edit/{id}`

## 2. 列表页能做什么

当前列表页支持：

- 查看任务列表
- 查看 cron 表达式的人类可读解释
- 创建任务
- 编辑任务
- 删除任务
- 测试执行
- 正式执行
- 刷新调度器
- 开启浏览器通知

## 3. 列表页几个关键按钮

### 应用

这个按钮非常重要。它不是“刷新页面”按钮，而是触发任务刷新，让新的任务配置真正进入调度器。

如果你：

- 新建了任务
- 编辑了 cron
- 改了启用状态

但没有点“应用”，后台未必会按新配置执行。

### 测试

测试执行的目标是：

- 验证模板能不能渲染
- 验证 Webhook 是否可用

它和正式执行的区别在于不会走完整采集流程。

### 执行

正式执行会尝试：

- 按任务配置处理公众号
- 更新文章
- 再发送消息

所以它更像“完整跑一遍任务”。

## 4. 浏览器通知是做什么的

列表页有浏览器通知开关。它属于页面增强能力，不影响后端任务本身。

开启后：

- 浏览器可以在检测到新文章时做标题闪烁或提示

不开启也不会影响后台消息任务正常跑。

## 5. 表单页字段怎么理解

当前表单的关键字段有：

- 任务名称
- 类型
- 消息模板
- WebHook 地址
- cron 表达式
- 公众号范围
- 状态

### 类型

当前主要有两种：

- `Message`
- `WebHook`

它们的核心差异在于模板和发送方式。

### 消息模板

页面内置了示例模板按钮。

当前逻辑会根据选中的公众号数量，自动给出两类示例：

- 单公众号模板
- 多公众号聚合模板

如果选择的是 `WebHook` 类型，还会给出 JSON 风格的示例模板。

### 公众号范围

可留空，也可多选。

留空通常表示：

- 对所有公众号生效

选择后则只处理指定公众号。

### cron 表达式

页面支持用选择器辅助填写，不要求手敲所有表达式。

## 6. 模板变量怎么理解

消息模板不是只替换一个标题字段，而是围绕“文章集合”和“公众号集合”渲染。

页面里的示例模板已经说明了两种典型上下文：

- 单公众号 + 多篇文章
- 多公众号聚合 + 每个公众号多篇文章

因此写模板时不要只假设有一个 `title` 变量，要按任务作用范围来写。

## 7. 一个正确的使用流程

如果你要建一个每日推送任务，正确顺序是：

1. 新建任务
2. 填任务名称
3. 选类型
4. 选 cron
5. 选公众号范围
6. 用示例模板起步
7. 提交保存
8. 回列表页点“应用”

最后这一步不能漏。

## 8. 常见问题

### 为啥任务保存了但没跑

优先检查：

- 是否点了“应用”
- 后端是否以 `-job True` 启动
- 任务状态是否启用
- cron 是否正确

### 测试成功但正式执行不对

说明模板和 Webhook 大概率没问题，优先查：

- 文章更新流程
- 公众号范围配置
- 微信登录状态

### 列表里显示 `WeekHook`

这是当前页面里的文案拼写问题，不影响功能本身，它实际表达的是 `WebHook` 类型。
