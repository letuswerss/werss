# 添加订阅

添加订阅页是把新公众号引入 WeRSS 的入口页。它既能手动新增，也能通过文章链接反查公众号信息，还集成了订阅列表的导入导出。

## 1. 访问路径

- `/add-subscription`

## 2. 这个页面能做什么

当前页面实际支持 4 类动作：

- 通过公众号关键字搜索候选公众号
- 通过微信文章链接提取公众号信息
- 手动填写并创建订阅
- 导出 / 导入订阅列表 CSV

它不是多标签页复杂表单，而是一个单页表单 + 两个辅助动作。

## 3. 页面结构

### 顶部操作区

页面头部通常有这几个按钮：

- 通过文章获取公众号信息
- 导出订阅列表
- 导入订阅列表

这里最有用的是“通过文章获取公众号信息”，因为它能减少手动输入错误。

### 主表单

主表单字段包括：

- 公众号名称
- 公众号 ID / 微信侧标识
- 头像地址
- 简介

页面会实时预览头像。

### 自动补全搜索

页面里还有一个搜索候选的自动补全能力：

- 输入关键字后调用公众号搜索接口
- 选择候选项后回填名称、ID、简介、头像

## 4. 最常用的操作方式

### 方式一：通过文章链接反查

这是最稳的一种：

1. 点“通过文章获取公众号信息”
2. 粘贴微信文章链接
3. 系统提取公众号信息
4. 自动回填到表单
5. 确认后提交

适合你已经看到了某篇文章，但系统里还没这个公众号的场景。

### 方式二：搜索后回填

如果你只有公众号名的模糊记忆：

1. 在搜索框里输入关键字
2. 从候选里选中目标公众号
3. 表单自动回填
4. 提交创建

### 方式三：完全手动录入

当搜索或文章提取都拿不到时，可以手动填：

- 名称
- 公众号 ID
- 头像 URL
- 简介

这种方式灵活，但最容易录错。

## 5. 导入导出怎么理解

页面上带了订阅导入导出功能，但当前实现是 CSV 方向，不是文档里常见那种“OPML 批量导入页”。

### 导出

点击导出后，会下载订阅列表 CSV。

### 导入

点击导入后，会选择本地 CSV 文件上传，并调用后端导入。

导入成功后，页面会提示：

- 新增数量
- 更新数量
- 跳过数量

然后跳回订阅管理页。

## 6. 提交成功后会发生什么

提交本页表单后：

1. 系统创建订阅记录
2. 跳回首页或订阅管理主链路
3. 后续由手动刷新或定时任务开始真正抓文章

也就是说：

- 添加订阅不等于立刻采到文章
- 文章还要靠后续采集链路补上

## 7. 常见问题

### 链接提取失败

优先检查：

- 是否真的是微信文章链接
- 链接是否完整
- 当前微信采集相关能力是否正常

### 选了搜索结果但字段不完整

因为候选结果本身可能只返回部分字段，必要时仍需手动修正。

### 添加成功后看不到文章

这是正常的。订阅创建只完成“建源”，采集还没发生或尚未完成。

## 8. 和其他页面的关系

- 添加完后，去 [订阅管理](./subscriptions.md) 看详情和手动刷新
- 文章真正进来后，去 [文章列表](./articles.md) 看结果

## 9. 当前实现边界

这页当前并没有真正做这些事：

- 多种标签页式批量导入流程
- OPML 导入流程
- 复杂的导入预览页

如果后面要补这些能力，应该基于当前代码再扩，不要按旧文档想象继续写。
