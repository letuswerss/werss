# 发展路线与待办事项

**维护说明：** 本项目的日常运维（包括 Issue/PR 处理、文档维护等）主要由 **Claude Code**、**Codex** 与 **GLM** 等工具链辅助实现；路线图中的研究课题与共建项目欢迎社区与课题组同步推进。

`README` 中提供了本路线图的**摘要表格**；完整的待办任务与进度以本文档为准。已完成项请标记为 `[x]` 或移动到“已完成”小节。

本文跟踪教学、研究与社区协作相关的待办；与 `README` 中的摘要表对照阅读即可。

## 1. 微信内容自动生成（支持特定主题）

**背景：** 本功能旨在根据指定的 **主题（Topic）** 自动生成适合微信公众号发布的文章草稿或结构化大纲。

| 状态 | 任务描述 |
| :--- | :--- |
| [ ] | 明确输入输出参数：包括主题、受众群体、字数要求、摘要/标签生成等，并对接现有文章入库流程。 |
| [ ] | 对齐采集与编辑流程：确保生成的内容能复用 `Article` 模型、共享数据库并支持审核状态流转。 |
| [ ] | 安全与合规保障：建立生成内容的免责声明、人工审核节点以及敏感词/事实核查策略。 |
| [ ] | 接口封装：将该功能封装为标准的 API 或脚本工具，并完善使用说明文档。 |
| [ ] | 样例准备：提供 1～2 个典型主题及预期输出示例，便于功能验收。 |

**建议标签（Issue）：** `skill`, `content-generation`, `wechat`

---

## 2. 智能标签系统：基于 GLiNER 的轻量化方案

**背景：** 探索比现有方案更轻量的文本处理方案。基于 **GLiNER** 类模型提取关键词与实体，并与当前的标签提取方案（如 KeyBERT、LLM）进行结果比对或级联增强。

| 状态 | 任务描述 |
| :--- | :--- |
| [ ] | 调研 GLiNER 在中文及多语种场景下的适用范围、推理成本与依赖库体积。 |
| [ ] | 建立评测标准：与现有标签提取行为对齐，制定离线评测指标（如 F1 值）和人工核对协议。 |
| [ ] | 实现后端集成：支持通过配置切换至 GLiNER 提取方法，并具备失败回退机制。 |
| [ ] | 资源文档化：说明模型来源、许可协议，并支持在 Docker 镜像中按需加载。 |
| [ ] | （可选）整理技术方案或报告素材，方便学术合作者复现研究结果。 |

**建议标签（Issue）：** `research`, `tags`, `gliner`, `performance`
