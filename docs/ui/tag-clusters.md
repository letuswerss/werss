# 标签聚类页面

## 面向谁

需要分析标签之间语义关系、发现潜在标签合并机会的运营人员。

## 解决什么问题

通过聚类算法将相似标签分组，查看每个聚类的成员标签和代表性标签（中心标签），并获取标签合并建议。

## 标签聚类列表页

**访问路径：** `/tag-clusters`

### 页面元素

| 区域 | 内容 |
|------|------|
| 页面标题 | 标签聚类 |
| 刷新按钮 | 重新加载聚类列表 |
| 重新分析按钮 | 触发 AI 重新聚类（耗时较长，弹窗显示进度） |
| 搜索框 | 按聚类名称、描述、中心标签 ID 搜索 |
| 聚类列表 | 展示聚类名称、描述、中心标签 ID、成员数量、版本号 |

### 列表字段说明

| 字段 | 说明 |
|------|------|
| 名称 | 聚类的名称，通常来自中心标签名称 |
| 描述 | 聚类的语义描述 |
| 中心标签 | 聚类的核心标签 ID，代表该组的主题 |
| 成员数量 | 该聚类包含的标签数量 |
| 版本号 | 聚类分析的版本，用于追踪更新 |
| 操作 | "查看" 按钮进入详情页 |

### 重新分析

点击"重新分析"后，AI 会对所有标签重新进行语义聚类，过程中显示弹窗提示（无法取消）。完成后自动刷新列表。

## 标签聚类详情页

**访问路径：** `/tag-clusters/{id}`

### 三个 Tab

#### 1. 概览（Overview）

展示聚类的基本信息：

| 字段 | 说明 |
|------|------|
| 聚类名称 | 该聚类的名称 |
| 成员数量 | 包含的标签数 |
| 中心标签 | 代表性标签名称和 ID |

下方有两个表格：

**成员标签列表：**

| 字段 | 说明 |
|------|------|
| 标签名称 | 成员标签的名称 |
| 标签 ID | 成员标签的 ID |
| 成员得分 | 该标签在聚类中的归属分数（分数越高越属于这个聚类） |

**合并建议（Merge Suggestions）：**

| 字段 | 说明 |
|------|------|
| 源标签 | 建议被合并的标签 |
| 目标标签 | 建议合并到的目标标签 |
| 总分 | 合并的语义相似度得分 |
| 原因 | 合并建议的理由说明 |

合并建议帮助运营者判断哪些标签语义重复，可以合并为同一个。

#### 2. 散点图（Scatter）

将聚类中的标签以向量形式投影到二维平面，支持 PCA / t-SNE / UMAP 三种降维方法。可调整参数：

| 参数 | 说明 |
|------|------|
| 降维方法 | PCA / t-SNE / UMAP |
| 包含边 | 是否显示标签之间的关系边 |
| 最小边权重 | 过滤低相似度的边 |
| 归一化 | 是否对向量做归一化处理 |

点击某个点会弹出提示显示标签名称。

#### 3. 网络图（Network）

展示聚类内标签的共现关系网络。节点大小反映成员得分，边的粗细反映标签间的相似度。

可用滑块调整"最小相似度阈值"，过滤掉相似度低于阈值的边，更清晰地展示核心关系。

同样支持点击节点查看标签名称。

## 理解 Merge Suggestion

Merge Suggestion 是 AI 分析后给出的标签合并建议：

- **源标签**（source_tag）：语义上可以被合并的标签
- **目标标签**（target_tag）：建议合并到的目标
- **总分**（score）：两个标签之间的语义相似度得分（原始值 / 10000 保留两位小数）
- **原因**（reason）：给出该建议的语义依据

运营者可以根据建议，手动将源标签的内容迁移到目标标签，然后删除源标签。

## 导出

详情页右上角提供"导出 JSON"按钮，将该聚类的完整信息（成员列表、合并建议、描述等）下载为 `tag-cluster-{id}.json` 文件。
