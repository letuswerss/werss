# 系统信息

系统信息页用于查看当前实例的运行状态、资源占用、微信登录状态、队列状态和任务调度状态。它更偏运维和排障，不是日常内容操作页。

## 1. 访问路径

- `/sys-info`

通常需要管理员权限。

## 2. 这个页面解决什么问题

当你遇到下面这些问题时，系统信息页通常是第一批排查入口：

- 后台任务没跑
- 微信登录好像失效了
- 文章补全队列卡住了
- 系统资源紧张
- 当前版本是否落后

## 3. 页面结构

当前页面主要包括 3 类信息。

### 系统资源

展示系统资源占用，例如：

- CPU
- 内存
- 磁盘
- 队列信息

资源卡片主要来自系统资源接口和单独的资源组件。

### 文章统计

如果接口返回了文章统计信息，会显示：

- 公众号总数
- 文章总数
- 无正文数量
- 有正文数量
- 已删除或异常数量

这部分很适合判断：

- 采集是否在正常积累数据
- 正文补全是否跟得上

### 系统与运行状态

包括：

- 操作系统信息
- Python 版本
- 系统运行时间
- API 版本
- 核心版本
- 最新版本
- 微信 token / 登录状态
- 队列状态

## 4. 页面里的几个关键信号

### 微信登录状态

如果微信未登录或登录即将失效，很多采集动作都会失败。出现采集异常时，先看这里，不要先查前端。

### 队列状态

它能帮助判断后台是否还有待处理任务，例如补全文、文章同步等。

### 最新版本 / 是否需要更新

这更多是运维提示，不代表必须立刻升级，但能帮助你识别当前实例是否明显落后。

## 5. 和设置页的关系

设置页更偏“控制”，系统信息页更偏“观测”。

可以这样理解：

- 设置页告诉你“系统应该怎么跑”
- 系统信息页告诉你“系统现在实际怎么跑”

两者要结合看，尤其是在任务异常时。

## 6. 常见排查路径

### 任务没跑

先看：

- 微信是否登录
- 队列是否卡住
- 调度器是否运行

### 文章少但订阅很多

先看：

- 文章统计
- 无正文数量
- 微信状态

### 页面能开但系统很慢

先看：

- 资源使用
- 队列积压

## 7. 适合谁看

- 管理员
- 部署与维护人员
- 排查线上问题的开发者
