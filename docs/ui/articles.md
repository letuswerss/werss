# 文章列表页

## 1. 页面作用

文章列表页是 WeRSS 的默认首页（`/` 或 `/articles`），是整个系统的"工作台"。用户在此完成文章的搜索、筛选、查看、批量操作、标签管理和 RSS 跳转等核心操作。

## 2. 入口路径

- 默认首页：`/`
- 显式路径：`/articles`
- 导航栏点击"文章"进入

## 3. 页面结构

页面分为三个区域：

### 3.1 筛选区（Filter Card）

位于页面顶部，包含：

- **关键词搜索**：输入框，支持按标题关键词搜索（多词 OR 匹配）
- **公众号筛选**：下拉框，选择"全部"或某个具体公众号
- **重置按钮**：清空筛选条件

筛选区下方还有一个**操作条**，包含：
- **隐藏 AI 筛选文章开关**：默认开启，隐藏 `ai_filter_status` 为 `hide` 的文章
- **AI 增强操作组**：文章过滤、标签提取、批量删除等操作入口

### 3.2 列表区（Main Table）

表格列：

| 列 | 说明 |
|---|---|
| 复选框 | 勾选文章，支持批量操作 |
| 标题 | 文章标题，点击可打开文章详情抽屉（Drawer） |
| 来源 | 公众号名称 |
| 标签 | AI 提取的标签，显示最多 3 个，超出显示 `+N` |
| 过滤标签 | AI 筛选状态：`keep`（保留）、`maybe`（待定）、`hide`（隐藏）。无状态显示 `—` |
| 发布时间 | 微信文章发布时间 |
| 操作 | 恢复隐藏文章（仅 `hide` 状态显示）、删除 |

**过滤标签与普通标签的显示差异：**

- **普通标签**（`tag_names` / `topic_names`）：以 `outline` 样式 Badge 显示，位于"标签"列，最多显示 3 个
- **过滤标签**（`ai_filter_status`）：以不同颜色 Badge 显示，位于"过滤标签"列：
  - `keep`：默认 outline 样式，表示保留
  - `maybe`：secondary 样式，表示待定
  - `hide`：destructive 样式，表示隐藏

### 3.3 批量操作区

位于列表区顶部右侧或操作条中，包括：
- 选中 N 篇文章后显示"已选 N 篇"
- 批量删除按钮（红色）
- AI 过滤/标签提取等批量操作下拉菜单

### 3.4 RSS 跳转区

页面右上角，包含"RSS"下拉按钮，支持生成当前筛选条件下的 RSS Feed：
- **ATOM**：Atom 格式
- **RSS**：RSS 2.0 格式
- **JSON**：JSON Feed 格式

链接规则：
- 选中公众号时：`/feed/mp/{mp_id}.{format}`
- 未选公众号（全部）：`/feed/all.{format}`

## 4. 常见操作

### 4.1 查看文章详情

点击文章标题，打开右侧抽屉（Drawer），显示：
- 文章标题、来源、发布时间
- 原文链接、上下篇导航
- 文章正文内容（已处理图片 URL）

### 4.2 切换文章状态

点击标题下方的状态 Badge（绿色=已发布，灰色=草稿），可切换 `status` 在 1（启用）和 2（禁用）之间切换。

### 4.3 AI 过滤

对文章进行 AI 二分类筛选（保留/隐藏）：

- **过滤选中**：对勾选的文章进行过滤
- **过滤当前页**：对当前页所有文章进行过滤
- **过滤未过滤**：对尚未进行 AI 筛选的文章进行过滤
- **过滤全库**：对数据库中所有文章进行过滤（需二次确认）

过滤过程中可暂停、继续、停止。完成后显示：隐藏数、保留数、待定数。

### 4.4 恢复隐藏文章

对于 `ai_filter_status` 为 `hide` 的文章，操作列显示"恢复"按钮，点击可恢复为 `keep` 状态。

### 4.5 标签提取

批量提取或重新提取文章标签：

- **提取选中**：对勾选的文章重新提取标签
- **提取当前页**：对当前页所有文章提取标签
- **补充缺失标签**：仅对尚未提取标签的文章提取（异步任务，有进度条）
- **提取全库**：对所有文章重新提取（需二次确认）

### 4.6 导出

点击右上角"导出"按钮，弹出导出配置弹窗，可导出选中文章或当前筛选结果。

## 5. 注意事项

- AI 过滤和标签提取为后台任务，中断后可在刷新页面后从 localStorage 恢复进度
- 删除为物理删除（`article.true_delete=true`），不可恢复
- 文章详情中的图片 URL 可能被处理过（添加 `/static/res/logo/` 前缀）
- `hideAiFiltered` 开关仅影响显示，不影响数据库

## 6. 常见报错 / 空态

| 场景 | 表现 |
|---|---|
| 无文章 | 表格显示"暂无数据" |
| 筛选无结果 | 表格显示"暂无数据"，检查筛选条件 |
| 标签提取失败 | Toast 提示错误信息，可重试 |
| AI 过滤中断 | 进度保存在 localStorage，刷新后可恢复 |
| 文章详情加载失败 | Toast 提示错误，详情抽屉关闭 |

## 7. 关联功能

- **RSS 输出**：`/feed/mp/{mp_id}.{format}` 或 `/feed/all.{format}`
- **标签管理**：标签在文章列表页提取，汇总到标签管理页面
- **AI 筛选**：筛选结果影响文章显示，过滤标签状态存储在 `article_ai_filters` 表
- **消息任务**：RSS 是被动的信息拉取，消息任务是主动的通知推送
- **导出**：文章可导出为 CSV/PDF/Markdown
