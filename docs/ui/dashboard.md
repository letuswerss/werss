# 仪表板

仪表板页用于给用户一个全局概览。它不是系统主工作台，但适合快速判断“系统最近有没有在正常工作”。

## 1. 访问路径

- `/dashboard`

## 2. 适合谁看

- 想快速看系统整体数据的人
- 管理员或运营人员
- 需要确认近期采集和标签趋势的人

## 3. 这个页面解决什么问题

用户不需要先进入文章列表、订阅页、标签页分别查看，而是在一个页面上先获得这些全局信号：

- 文章总量
- 订阅总量
- 今日新增
- 近一周新增
- 文章来源分布
- 热门标签或关键词趋势

## 4. 页面结构

当前页面主要由三部分组成：

### 统计卡片

展示系统当前最重要的几个总量指标。

典型包括：

- 总文章数
- 总订阅数
- 今日新增
- 近 7 天新增

### 图表区

当前仪表板引入了较重的图表库，并做了延迟加载与错误兜底。图表主要用于展示：

- 来源分布
- 关键词分布
- 标签趋势

这也是为什么仪表板在实现上比一般“卡片页”复杂。

### 近期趋势与统计明细

让用户看到过去一段时间的数据变化，而不是只看静态总数。

## 5. 数据来源

优先来源：

- 仪表板专用接口 `getDashboardStats`

回退来源：

- 文章列表接口
- 订阅列表接口

当前实现里，如果专用接口失败，会退回到前端自己拉文章和订阅数据后做本地统计。这意味着：

- 页面能更稳地展示
- 但在数据量大时，回退逻辑会更重

## 6. 使用时怎么理解这些数字

### 今日新增

表示今天新进入系统的文章量，用于看采集是否在工作。

### 总订阅数

表示系统当前管理的公众号来源数量。

### 关键词趋势

它更多是“内容信号”而不是严格的数据指标。适合观察近期关注点，不适合当精确分析结论。

## 7. 常见问题

### 图表加载失败

当前页面有单独的错误边界和懒加载兜底。如果看到“图表加载失败”，通常是图表库加载异常，而不是整个仪表板接口都挂了。

### 数字和列表页不完全一致

可能原因：

- 仪表板走的是聚合统计接口
- 或者走了回退统计逻辑
- 某些页面默认隐藏了 AI 过滤文章

## 8. 和其他页面的关系

- 想看细节：去文章列表页
- 想改来源：去订阅管理页
- 想看标签整理：去标签页
- 想看热点事件：去热点发现页

所以仪表板更像“概览页”，不是操作页。
